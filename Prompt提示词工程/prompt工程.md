# Prompt工程

核心定义：Prompt（提示词）是用户与AI模型沟通的指令，Prompt工程是通过优化提示词设计，让AI精准输出符合需求的结果，核心是「清晰、具体、有引导性」，适配OpenAI、Ollama等各类大模型调用。

## 一、Prompt核心设计原则（必记）

### 1. 清晰具体，拒绝模糊

- 错误：帮我写个代码。

- 正确：帮我写一段Python代码，用for循环输出1到5的数字，注释清晰，简洁易懂。

核心：明确「任务类型+具体要求+输出格式」，减少AI猜测空间。

### 2. 角色设定（对应system角色）

在Prompt开头设定AI角色，明确其身份和能力，让输出更贴合场景。

示例：「你是一个Python编程新手导师，用最通俗的语言解释代码，避免专业术语，步骤清晰。」

### 3. 使用分隔符去表明输入的不同部分

通过**分隔符**（三引号、[]等）明确划分 Prompt 中不同功能的文本，让模型精准区分「指令」和「待处理内容」，避免模型混淆指令与输入，大幅提升输出准确性。

示例：请用20个字符总结由三引号分割的文本。 """""" 在此插入待总结的长文本内容 """"""

### 4. 对任务指定步骤

将**可拆分的复杂任务拆解为一系列明确、有序的步骤**，让模型按步骤执行，降低理解难度，大幅提升输出的准确性和完整性，避免模型遗漏关键环节。

比如：第一步干什么、第二步干什么.....

### 5. 提供例子

这个技巧的本质是**少样本学习（Few-shot Learning，也叫上下文学习 In-context Learning）**：

- 核心逻辑：在提示词中给大模型提供**1~5 个「输入→输出」的标准示例**，让模型快速学习任务的规则、格式、风格，再让它按示例模式生成新内容。

### 6. 使用参考文本

降低模型一本正经的胡说，降低模型的幻觉。经典的知识库用法，也就是RAG。

## 二、Zero-shot（零样本）与 Few-shot（少样本）思想

### 1. Zero-shot Learning（零样本学习）

**核心思想**：不给大模型任何示例，仅通过**自然语言描述任务要求**，让模型直接完成任务。

- 底层逻辑：依赖大模型在预训练阶段学到的海量通用知识与世界常识，让模型仅凭任务描述理解需求、自主推理并生成结果。
- 本质：用「语言指令」替代「示例」，考验模型的泛化能力与指令理解能力。
- 类比：就像给学生一张全新题型的试卷，只说 “按要求做题”，不给任何例题，学生靠自己的知识储备完成。

### 2. Few-shot Learning（少样本学习，也叫上下文学习 In-context Learning）

**核心思想**：给大模型提供**少量（通常 1~5 个）「输入→输出」的标准示例**，让模型快速学习任务的规则、格式、风格，再按示例模式完成新任务。

- 底层逻辑：大模型通过上下文示例学习任务的隐性模式，用 “示例” 这种最精准的语言传递需求，大幅降低指令理解的偏差。
- 本质：用「示例」给模型做 “任务引导”，让模型快速适配自定义规则，提升输出准确率。
- 类比：给学生 2~3 个同题型的例题，再让他做新题，学生通过例题掌握解题方法，再完成新任务。

## 三、使用案例

Few-shot Learning少样本学习案例：

```java
from openai import OpenAI
import os
# 1. 初始化客户端（替换为你的API Key）
client = OpenAI(
    # 如果没有配置环境变量，请用阿里云百炼API Key替换：api_key="sk-xxx"
    api_key=os.getenv("DASHSCOPE_API_KEY"),
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
)

# 2. 定义Few-shot示例数据（核心：输入文本 → 分类结果）
examples_data = {
    "新闻报道": "今日，央行发布公告宣布降准0.5个百分点，释放长期流动性约1万亿元",
    "公司公告": "ABC公司今日发布公告称，已成功完成对XYZ公司股权的收购，交易金额达50亿元",
    "财务报道": "公司资产负债表显示，公司偿债能力强劲，现金流充足，为未来投资和扩张提供了坚实的财务基础",
    "分析师报告": "最新的分析报告指出，可再生能源行业预计将在未来几年经历持续增长，投资者应该关注这一领域的投资机会"
}

# 3. 定义待分类的测试文本
questions = [
    "小明喜欢小新哟",  # 非金融文本，应分类为「不清楚类别」
    "证监会发布最新监管政策，强化上市公司信息披露要求",
    "特斯拉发布2025年Q1财报，净利润同比增长20%，超市场预期",
    "中金公司发布研报，上调贵州茅台目标价至2000元，维持买入评级",
    "腾讯控股发布公告，拟回购不超过10亿美元的公司股份"
]

# 4. 构造messages列表（符合OpenAI对话格式）
messages = [
    {
        "role": "system",
        "content": "你是金融专家，将文本分类为['新闻报道', '财务报道', '公司公告', '分析师报告']，不清楚的分类为'不清楚类别'，只返回分类结果，不要额外解释"
    }
]

# 5. 循环添加Few-shot示例（user输入 → assistant输出）
for key, value in examples_data.items():
    messages.append({"role": "user", "content": value})
    messages.append({"role": "assistant", "content": key})

# 6. 批量调用API完成分类
for q in questions:
    # 每次请求复制messages，避免历史对话累积
    current_messages = messages.copy()
    current_messages.append({"role": "user", "content": q})

    response = client.chat.completions.create(
        model="qwen3.5-plus",
        messages=current_messages,
        temperature=0  # 温度设为0，保证分类结果稳定
    )
    print(f"文本：{q}")
    print(f"分类结果：{response.choices[0].message.content}\n")
```
